Registering Dynamic Hosts in DNS¶

ddns-confgen -q

key "ddns-key" {
    algorithm hmac-sha256;
    secret "zxHaXudonbUFek9EFGPCbTwgIT/BCZJEOTWEDtaMDEY=";
};

Copy the key declaration and paste into a new file named /etc/bind/ddns-update.key. Take note of key's label, which will be referenced in named and dhcpd configuration.

Ensure that the key is readable by root/bind, but not by any other users on the server

chown root:bind ddns-update.key
chmod 660 ddns-update.key

Configure Dynamic Zones in BIND¶

Enable updates by adding a per-zone update-policy in /etc/bind/named.conf.local. Since the policy statement refers to the key by name, you will also need to include contents of the newly generated key before 

include "/etc/bind/ddns-update.key"; // (1)!

zone "example.pi" {
    type master;
    file "db.example.pi";
    update-policy { 
        grant ddns-key zonesub ANY;
    }; // (2)!

zone "168.192.in-addr.arpa" {
    type master;
    file "db.192.168";
    update-policy { 
        grant ddns-key zonesub ANY;
    }; // (3)!

};
  1. Include the newly generated key before the zone declaration.
  2. Add an update-policy to each dynamic zone, referencing the key by its label and listing permitted record types.
